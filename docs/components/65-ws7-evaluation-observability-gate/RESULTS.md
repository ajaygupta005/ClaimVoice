## Observability Fields

Each turn trace includes:
- `turn_id` — unique UUID hex per scenario run
- `scenario_id` — maps back to dataset case ID
- `intent` — routing outcome
- `grounded` — hallucination guard result
- `total_ms` — end-to-end pipeline latency
- `stages[]` — identify / understand / tool / guard / respond with status
- `tools[]` — tool name, args, result summary, data_source, error_code
- `guard` — passed + reason
- `answer_snippet` — first 200 chars of answer
- `tool_mode` / `composer_mode` — runtime configuration

When `LANGFUSE_SECRET_KEY` and `LANGFUSE_PUBLIC_KEY` are set, each turn
emits a Langfuse trace with tool spans for debugging.
